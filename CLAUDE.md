## Project Overview

This is a minimal **Nuxt 4** full-stack Vue application for teacher scheduling. The project uses:
- **Vue 3** with Composition API
- **Nuxt 4** - meta-framework with built-in SSR, file-based routing, and auto-imports
- **@nuxt/ui** - Headless UI component library
- **Tailwind CSS** - utility-first CSS framework
- **TypeScript** - for type safety
- **pnpm** - package manager

## Development

### Install & Run

```bash
pnpm install    # Install dependencies
pnpm dev        # Start dev server at http://localhost:3000
```

### Build & Preview

```bash
pnpm build      # Build for production (.output directory)
pnpm preview    # Preview production build locally
pnpm generate   # Generate static site (if needed)
```

### Project Structure

- **`app/`** - Application root
  - `app.vue` - Root component (entry point for rendering)
  - `assets/css/main.css` - Global styles (Tailwind)
- **`nuxt.config.ts`** - Nuxt configuration (modules, css imports, etc.)
- **`tsconfig.json`** - References generated Nuxt tsconfig files in `.nuxt/`

Nuxt auto-discovers:
- **`pages/`** directory for file-based routing
- **`components/`** directory for auto-imported Vue components
- **`composables/`** directory for auto-imported composable functions
- **`server/`** directory for server routes and middleware

## Architecture Overview

### Auth (Better Auth)
- Server config: `server/utils/better-auth.ts` — drizzle adapter, argon2, bearer plugin
- Catch-all route: `server/api/auth/[...all].ts` → `auth.handler(toWebRequest(event))`
- Client singleton: `app/utils/auth-client.ts` — `createAuthClient()` from `better-auth/vue`
- Composable: `app/composables/useAuth.ts` — wraps authClient; provides login/register/logout/session

### Database (Drizzle ORM + PostgreSQL)
- Connection: `server/db/index.ts` — `getDatabase()` singleton, reads `DATABASE_URL` env var directly
- Schema: `server/db/schema.ts` — users, sessions, accounts, verifications tables
- Queries: `server/db/queries/users.ts` — typed CRUD for users table
- Migrations: `server/db/migrations/` — generated by drizzle-kit

### Pages & Layouts
- `app/layouts/default.vue` — UHeader (nav + login/register buttons) + UMain + UFooter
- `app/pages/auth/login.vue` — login form (UCard, UFormField, UInput, UButton, UAlert)
- `app/pages/auth/register.vue` — registration form (same components + name field)
- `app/pages/index.vue` — home page (placeholder)

## Import Alias Rules

- `~/` — Nuxt alias (project root), use in `app/` files
- `~~/` — Nitro alias (project root), use in `server/` files
- Vitest config: `~` → project root via `vitest.config.ts` resolve alias
- Do NOT cross aliases — server files using `~/` will fail at build time

## Testing Structure

- `tests/unit/` — isolated unit tests with mocks (no real DB)
- `tests/integration/` — multi-function tests that hit the real database
- Run all: `pnpm test` | DB tests only: `pnpm test:db`
- Vitest env: `node` (no DOM), globals enabled, loads `.env` via `dotenv.config()`
- Mock Nuxt globals in unit tests: `vi.stubGlobal('navigateTo', vi.fn())`
- Mock better-auth client: `vi.mock('~/app/utils/auth-client', () => ({ authClient: { ... } }))`

## Key Gotchas

- Better Auth manages its own session/account tables — do not manually insert into them
- `navigateTo` is a Nuxt auto-import global — never import it; mock with `vi.stubGlobal` in tests
- `better-auth/vue` must be in `vite.optimizeDeps.include` in `nuxt.config.ts` (already done)
- DB connection in `server/`: always use `getDatabase()`, reads `process.env.DATABASE_URL` directly (not runtimeConfig)
- Auth pages use `definePageMeta({ auth: false })` to opt out of future route guards
