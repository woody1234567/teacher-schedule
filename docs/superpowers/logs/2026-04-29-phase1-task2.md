## 執行過程中遇到的問題

### 1. Drizzle-kit 配置版本不匹配
**問題描述：**
執行 `pnpm db:migrate` 時，drizzle-kit 報告 config 格式無效：
- 需要 `dialect` 參數（非 `driver`）
- 需要使用 `url` 而不是 `connectionString`

**解決方法：**
更新 drizzle.config.ts 以符合 Drizzle Kit v0.31.10 的最新配置格式：
```typescript
export default {
  schema: './server/db/schema.ts',
  out: './server/db/migrations',
  dialect: 'postgresql',  // 改自 driver: 'pg'
  dbCredentials: {
    url: process.env.DATABASE_URL,  // 改自 connectionString
  },
}
```

**結果：** ✅ 已解決

### 2. 資料庫連接不可用（開發環境）
**問題描述：**
執行 `pnpm db:migrate` 時因本地 PostgreSQL 未運行而失敗。

**解決方法：**
使用 `drizzle-kit generate` 替代 `migrate`，該命令只生成 SQL 遷移文件而不需要實際的數據庫連接。生成的 SQL 可稍後在數據庫可用時執行。

**結果：** ✅ 已解決 - 遷移文件成功生成

## 所使用的解決方法

### 1. Drizzle ORM Schema 定義
創建 `server/db/schema.ts` 定義三個主表：

- **users 表**（9 列）
  - id (text, primary key)
  - name, email, password_hash, image
  - email_verified (boolean, 默認 false)
  - metadata (json, 用於存儲額外用戶信息)
  - created_at, updated_at (timestamps with defaults)
  - 約束：email 唯一

- **sessions 表**（7 列）
  - id (text, primary key)
  - user_id (text, 外鍵)
  - token (text, 唯一)
  - expires_at (timestamp, 會話過期時間)
  - ip_address, user_agent (用於會話追蹤)
  - created_at (timestamp)

- **accounts 表**（9 列）
  - 用於 OAuth 多提供商支持
  - id, user_id, provider, provider_account_id
  - accessToken, refreshToken, expiresAt
  - created_at, updated_at

### 2. 數據庫連接函數
創建 `server/db/index.ts` 提供：

```typescript
export function getDatabase()  // 單例模式獲取數據庫連接
export async function testConnection()  // 測試連接並返回結果
export default getDatabase()  // 默認導出
```

### 3. 測試框架配置
- 創建 `vitest.config.ts` 配置測試環境
  - 使用 node 環境（非 jsdom）
  - 啟用全局 API
  - 配置代碼覆蓋率（V8 provider）
  - 配置路徑別名 (~)

- 創建 `server/db/index.test.ts` 測試數據庫連接
  - 測試 testConnection() 返回預期的屬性
  - 驗證返回對象有 success 和 message 屬性

### 4. 依賴安裝
```bash
pnpm add -D vitest @vitejs/plugin-vue @vitest/ui
```

### 5. Package.json 更新
添加新的 npm 腳本：
- `test` - 運行所有測試
- `test:ui` - 運行測試及 UI 查看器
- `test:db` - 只運行數據庫相關測試

### 6. 數據庫遷移生成
```bash
pnpm exec drizzle-kit generate --config drizzle.config.ts
```

生成的 SQL 遷移文件 `server/db/migrations/0000_petite_wendell_vaughn.sql` 包含：
- CREATE TABLE "users"（9 列，包括唯一約束）
- CREATE TABLE "sessions"（7 列，包括唯一約束）
- CREATE TABLE "accounts"（9 列）

## 架構決策說明

### 為什麼使用 Drizzle ORM？
- **類型安全**：完整的 TypeScript 支持
- **簡潔的 API**：相比 Sequelize 或 TypeORM，代碼更簡潔
- **良好的遷移管理**：drizzle-kit 提供開箱即用的遷移工具
- **Nuxt 適配**：與全棧 Nuxt 應用集成良好

### 為什麼包含 accounts 表？
- **OAuth 支持**：為 Phase 2+ 的 Google/GitHub OAuth 預做準備
- **多提供商**：一個用戶可關聯多個 OAuth 提供商
- **靈活性**：支持未來添加更多認證提供商

### 為什麼選擇文本型 id？
- **UUID 友好**：應用層生成 UUID，而非數據庫自增
- **分佈式友好**：便於未來的數據庫分片或複製
- **安全性**：避免序列號產生的信息洩露
